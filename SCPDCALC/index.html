STANDARD FORMS

Contains information regarding standard forms of certain equations or functions.

Linear Equations are written in standard form as ax + b = 0.

Quadratic Equations are written in standard form as ax2 + bx + c = 0.

The standard form of Quadratic Functions are different from Quadratic Equations: ax2 + bx + c is only the general form. The standard form for Quadratic Functions is a(x - h)2 + k. Note that including f(x) or y makes it equal to the form y = a(x - h)2 + k.

Warning Information

All inputs are evaluated before the operations are run. So if you input 5 + 7, the evaluated input will be 12.

Because of the computer's understandable inability to handle really big numbers, 1e+308 is the largest finite number you could input.

Many times the given trigonometric functions will output the incorrect answer. For example, inputting Math.tan(90) will NOT be evaluated as either NaN or undefined or even Infinity.

All given outputs involving Pi or Euler's Number will never be exact, and may sometimes be incorrect.

If you input a number large enough, say, 3141596253589831416, then it will automatically be rounded off by a certain place, such as 3141596253589831000.

All numbers larger than 1e20 will output scientific notation. For example, 1000000000000000000000000 will be read as 1e+24.

Inputting commas that are not in functions will result in errors.

Answers that result as Undefined, such as 0/0, will output as NaN, or "Not A Number".

Directly inputting imaginary or complex numbers, such as 2i or 3 - i, will result in errors. JavaScript does not yet support imaginary numbers.

Sometimes the outputs will be labeled different from expectations. For example, in the Square Calculator, instead of the label "Diameter of Circumscribed Circle", it is labeled as "Diagonal" instead.

Sometimes, inputting decimal numbers result in the wrong outputs. For example, inputting the square root of 2 in the Square Calculator will result in the Area being 2.0000000000000004 instead of 2.

Some calculators contain a LOT of extensive information. For example, the Square Calculator will not only calculate for the square, but also for its circumscribed and inscribed circles. This may be useful when dealing with certain problems.

Inputting negative numbers in geometric calculators will negate the outputs. Signs don't affect the operations.

For calculators involving variables, standard form is observed when inputting numbers.

Math.round is used in the Numeric Calculator conditions of Is Odd (Integered) and Is Even (Integered). Is Odd and Is Even will show False if a decimal is inputted.

Inputting variables will output errors unless the variable is, by chance, defined. But only when the variable is global, and its definition is a number.

In the Numeric Calculator, trigonometric functions accept the input in radians, not degrees. If you input 1, then the conditions for the number being Composite or Prime will return true due to 1's properties, even though it is debatable. (The function used is notably slow, so it may take a while for big numbers.)

In the Quadratic Equation Calculator, inputting zero for value A will result in an error. Instead, input B and C as A and B in the Linear Equation Calculator.
